# Как я сделал российское зеркало своего сайта, чтобы пользователи могли учиться без VPN

Привет! Я развиваю два бесплатных сайта для практики SQL - [sqltest.online](https://sqltest.online) и [sqlize.online](https://sqlize.online). Суть простая: хочешь научиться писать запросы к базам данных - вот тебе живой MySQL, PostgreSQL, Oracle и другие прямо в браузере, без установки чего-либо.

И вот однажды начали приходить сообщения: «Сайт не открывается. Только через VPN».

Поначалу думал - единичные случаи. Но сообщений становилось всё больше. Оказалось, больше 30% моей аудитории из России, и для них зайти на sqltest.online превратилось в квест.

## Что я придумал

Переносить весь сервер в Россию - дорого и сложно. Там крутится целый зоопарк баз данных: MySQL, PostgreSQL, MariaDB, Oracle, MS SQL Server - каждая в своём Docker-контейнере. Это не маленькая виртуалка, железо нужно серьёзное.

Поэтому я пошёл на хитрость: сделал **гибридную схему**.

**Фронтенд** (то, что видит пользователь - интерфейс, страницы) - вынес на российский хостинг, зарегистрировал домен `sqltest-online.ru`. Сайт написан на чистом PHP без фреймворков и нативном JavaScript, так что развернуть его на бюджетном хостинге - дело нескольких часов.

Для российского зеркала я выбрал хостинг от [SpaceWeb](https://sweb.ru/?utm_term=paveavec) - у них отличные цены на бюджетные тарифы и классная поддержка, которая реально помогает.

**Бэкенд** (сами базы данных, где выполняются запросы) - остался на прежнем сервере в Германии.

Работает это так: ты нажимаешь «Выполнить запрос» - твой браузер стучится на `.ru`-сервер в России, а тот уже сам, по-тихому, пересылает запрос в Германию и возвращает тебе результат. Ты это не замечаешь, но страница открывается без VPN.

```
Ты (РФ) → сайт в России → сервер в Германии → база данных → результат тебе
```

Красиво? Мне кажется, да.

## Как деплой чуть не убил идею

Залил код на хостинг - всё отлично. Запускаю установку зависимостей через Composer (это такой менеджер пакетов для PHP) - и тишина. Процесс завис. Совсем.

Проверил через `curl` - оказалось, сервер в России просто не может достучаться до репозитория пакетов на packagist.org. Таймаут.

В 2026 году это уже норма: зарубежные сервисы или блокируют запросы из России, или российская сторона что-то режет - в любом случае пакеты не качаются.

Написал в поддержку хостинга. И вот тут мне повезло: вместо дежурного «проблема на вашей стороне» ребята реально разобрались, подтвердили проблему и дали адрес прокси.

Одна строчка в терминале - и Composer всё скачал:

```bash
export HTTPS_PROXY=http://адрес-прокси
composer install
```

Итого на решение ушло меньше часа.

**Лайфхак на будущее:** если деплоите PHP-проект на российский хостинг и Composer завис - первым делом проверьте, открывается ли packagist.org вообще. Если нет - сразу идите в поддержку за прокси, не тратьте время на отладку самого Composer.

## Что в итоге

Зеркало работает. Жалоб на «не открывается» стало заметно меньше.

Есть ли у схемы слабое место? Да - если сервер в Германии ляжет, оба домена упадут одновременно. Резервного бэкенда нет: сайт бесплатный, живёт на небольшой рекламе и донатах, на второй сервер пока не хватает. Работаю над этим.

Если вы хотите попрактиковаться в SQL - заходите на [sqltest-online.ru](https://sqltest-online.ru) или [sqltest.online](https://sqltest.online), всё бесплатно.
